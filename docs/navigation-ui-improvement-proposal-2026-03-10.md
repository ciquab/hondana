# アプリ内導線 改善提案（ボタン配置・メニュー配置・情報配置）

作成日: 2026-03-10
更新日: 2026-03-10（Step 3.9 実装反映）
対象: 保護者画面 / キッズ画面 全体導線

## 1. 目的

- 主要タスク（記録する / 見守る / 反応する）までのクリック数と迷いを減らす。
- 画面ごとの「最優先アクション」を視覚的に一貫させる。
- 情報の表示順を「判断に必要な順番」に寄せ、読み飛ばしを防ぐ。

## 2. 現状導線の要約

### 2-1. 保護者導線

- ダッシュボードに「今月サマリー」「よく使うメニュー」「今日のやること」「子ども一覧」が並ぶ。
- 子ども詳細で、記録追加・おすすめ提案・ミッション設定・記録一覧を操作。
- 記録詳細でコメント / リアクション / 編集 / 削除を行う。

### 2-2. キッズ導線

- ホームに 2x2 のナビカード（記録・本棚・カレンダー・メッセージ）。
- 各サブページにはテキストリンクで「ホームへ戻る」。
- 記録作成画面は検索・スキャン・手入力を同画面で切替。

## 3. 課題（導線・配置観点）

### 課題A: 「戻る/移動」のUIが画面ごとに弱く、位置が散らばる

- キッズ各ページで戻りがテキストリンクのみで、タップ対象が小さい。
- 保護者画面も「戻る」が見出し内テキストに埋もれやすい。

### 課題B: 主要CTAが画面ごとに不統一

- 「記録を追加」「コメントする」「読んだ！」など重要操作の見た目・優先度がページ間で揃っていない。
- 同一目的（次の行動へ進む）でも、リンク・ボタン・カードが混在し視線移動コストが高い。

### 課題C: 情報の表示順が「概要→次アクション」ではなく分散

- 保護者ダッシュボードで、最初に実行すべき「今日のやること」がサマリーやメニューの後段にある。
- 子どもホームでも、未読メッセージ・進行中ミッションなど「今すぐ反応が必要な情報」がナビカード後方にある。

### 課題D: 複数子ども運用時に、メニュー遷移先が子どもコンテキストを保持しない

- ダッシュボード上部の共通メニューは家族/子ども管理系で、子ども個別タスクへのショートカットが弱い。
- 子ども別に「今なにをすべきか」がタブに入らないと見えにくい。

### 課題E: 一部画面で「空状態」の次アクション誘導が弱い

- メッセージ0件・記録0件の導線はあるが、同じ導線設計（色/文言/ボタン位置）が揃っていない。
- 初回ユーザーが次にやることを学習しづらい。

## 4. 改善提案（優先度つき）

## P0（最優先）

### 提案1: 画面上部に「固定ナビ行」を導入（戻る + ページ名 + 主CTA）

- 共通レイアウトとして以下を統一:
  - 左: 戻る（アイコン付きボタン）
  - 中央: ページタイトル
  - 右: 主CTA（例: 記録する / コメントする）
- 効果:
  - 位置学習が起き、どの画面でも迷いが減る。
  - スマホ片手操作で戻るミスを減らせる。

### 提案2: 保護者ダッシュボードの表示順を「行動優先」に再構成

推奨順:
1. 今日のやること（最上段）
2. 子ども別クイックアクション（記録追加 / 最新記録へ）
3. 今月サマリー
4. よく使うメニュー（設定系）

- ポイント:
  - KPI表示（今月◯冊）より先に「今やる操作」を置く。
  - 設定系リンクは下段へ寄せる。

### 提案3: キッズホームの情報順を「通知→行動→成果」に変更

推奨順:
1. 未読メッセージ・期限付きミッション（アラート帯）
2. 2x2ナビカード
3. さいきんよんだ本
4. バッジ・おすすめ

- 未読がある場合、メッセージカードを先頭表示し CTA を「よみにいく」にする。

## P1（高）

### 提案4: 主要ボタンのデザインシステムを3種類に整理

- Primary（1画面1つ）: もっとも実行してほしい操作
- Secondary（2〜3個）: 補助操作
- Tertiary（テキストリンク）: 戻る、詳細、ヘルプ

ルール:
- 1画面内に Primary は 1個まで。
- Primary は最下部固定 or ヘッダ右上固定のどちらかに統一。
- キッズ画面は色数を抑え、強調色を Primary のみへ集中。

### 提案5: 子ども詳細（保護者）の「タスク面」を上段固定

- 現状の「保護者メニュー」「ミッション」「本棚」「ジャンル」「記録一覧」を、以下へ再編:
  1. 本日のアクション（コメント未対応、記録追加）
  2. ミッション進捗
  3. 分析（ジャンル/冊数）
  4. 記録一覧
- 記録一覧はタブ内で「未コメント / 最近 / ステータス別」を切替可能にする。

### 提案6: 空状態テンプレートを共通化

空状態の必須要素:
- イラスト or アイコン
- 状態説明（1文）
- 次アクション（1つ）
- 代替アクション（必要なら1つ）

これをキッズメッセージ、子ども詳細0件、カレンダー0件などへ統一適用。

## P2（中）

### 提案7: ボトムナビ（キッズ）を検討

- 対象ページ: ホーム / 記録 / カレンダー / メッセージ
- 常時表示で戻る回数を削減。
- 現在地を強調し、未読バッジをナビアイコンにも表示。

### 提案8: 保護者向け「子どもコンテキスト維持」導線

- ダッシュボードで子ども選択後、関連操作（記録追加・最新記録）を同カード内に配置。
- 直近閲覧 childId を保存し、詳細画面遷移時の初期表示に反映。

## 5. ページ別 改善具体案

### 5-1. ダッシュボード

- 「今日のやること」をヘッダ直下へ。
- 子どもタブカード内に「＋記録」「最新を見る」を横並び追加。
- 「よく使うメニュー」は折りたたみ（初期は閉じる）で情報密度を調整。

### 5-2. 子ども詳細（保護者）

- ページ上部に sticky 操作帯（記録追加 / おすすめ送る）。
- 記録一覧カードの右端 CTA を「コメントする」「詳細を見る」で明確分離。
- ジャンル内訳は初期折りたたみ、必要時のみ展開。

### 5-3. 記録詳細（保護者）

- 表示順を「本情報 → 親の操作（コメント/リアクション）→ メモ/履歴」へ変更。
- 「削除」は画面最下部の危険エリアへ隔離し、誤タップ防止。

### 5-4. キッズホーム

- 未読メッセージがある場合は最上部に固定通知。
- ナビカード4枚のうち、利用頻度の高い「きろくをつける」を左上固定維持。
- 最近読んだ本の「つづきをきろく」CTAをカード化。

### 5-5. キッズ記録作成

- 入力モード（かんたん/くわしく）は画面上部で sticky。
- 検索失敗時は既に良い導線があるため、ボタン順だけ統一:
  1) 別ワード検索 2) バーコード 3) 手入力
- 保存CTAを常に画面下部固定して完了率を上げる。

### 5-6. キッズメッセージ

- 「よんだ！」ボタンを各カード右下固定にし、日付は右上維持。
- 未読カードをリスト先頭へ自動ソート。
- 0件時CTAの文言をホームと統一。

### 5-7. キッズカレンダー

- 月移動ボタンを上部固定し、当月へ戻るボタンを追加。
- 日付セルの情報優先度を「スタンプ > 冊数」に変更（子ども理解優先）。

## 6. 期待効果（KPI）

- 記録作成完了率: +8〜15%
- 保護者コメント実行率: +10〜20%
- キッズメッセージ既読化率: +15%
- 主要タスク到達時間（ホーム→記録開始）: -20%

## 7. 実装ロードマップ案

1. Sprint 1（1週間）
   - 固定ナビ行
   - ダッシュボード表示順変更
   - キッズホーム情報順変更
2. Sprint 2（1週間）
   - ボタン階層統一（Primary/Secondary/Tertiary）
   - 空状態テンプレート共通化
3. Sprint 3（1〜2週間）
   - 子ども詳細のタスク面再編
   - キッズボトムナビ PoC

## 8. 検証方法（A/B または段階リリース）

- 計測イベント例:
  - dashboard_action_click
  - kid_home_nav_click
  - kid_message_mark_read
  - record_create_start / record_create_submit
- 成功判定:
  - 2週間平均で KPI が閾値を超える
  - 離脱率悪化がないこと

## 8.5 実装反映状況（Step 3.9 / 2026-03-10）

- ✅ Sprint A 実施済み
  - 固定ナビ行（主要画面）
  - ダッシュボード表示順の行動優先化
  - キッズホーム情報順の再編（通知先頭）
- ✅ Sprint B 実施済み
  - ボタン階層（Primary/Secondary/Tertiary）の統一
  - 空状態テンプレートの共通化（メッセージ/カレンダー/子ども詳細）
- ✅ Sprint C 実施済み
  - 導線KPIイベント計測の初期実装（送信ユーティリティ + API受け口 + 主要導線への埋め込み）

## 9. まとめ

- 改善の軸は「情報を増やす」ではなく「順番と強弱を揃える」こと。
- まずは P0（固定ナビ・表示順再編）を行い、次にボタン階層統一へ進むのが費用対効果が高い。
