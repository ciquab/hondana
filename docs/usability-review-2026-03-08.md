# ほんだな ユーザー目線評価（2026-03-08）

## 前提
- モバイル幅（390px）でログイン画面・こどもログイン画面の実機表示を確認。
- 認証後画面はコードレビュー中心で評価。

## 総評
- デザイン全体は親しみやすく、子ども向け/保護者向けのトーン分けは概ねできている。
- 一方で、**子どもモードの漢字混在**、**用語の揺れ（読書中/よんでる等）**、**編集・削除導線の不足** がUX上の主要課題。

## 見やすさ
### 良い点
- カードUI・絵文字活用・余白設計により情報のグルーピングが明確。
- キッズホームの4分割ナビは用途が直感的。

### 気になる点
- 親向け画面は情報密度が高く、1画面内のアクションが多い（特にダッシュボード上部）。
- 子ども本棚は表紙風UIが魅力的だが、タイトル文字が小さく2行固定のため長文判読が難しい。

## 文字の読みやすさ
### 良い点
- 大見出し・主要ボタンはおおむね十分なサイズ。
- 子ども向け画面はひらがな中心の文言が多く、心理的ハードルが低い。

### 気になる点
- 10px前後のテキスト（本棚タイトル等）はスマホで視認性が低い。
- 一部で背景色と文字色が淡く、視認性が下がる箇所がある（補助情報の薄いグレー文字）。

## 用語・表現の統一感
### 指摘
- ステータス表現が親と子で意図的に違う一方、同一モード内でも揺れがある。
  - 例: 保護者向けは「読みたい / 読書中 / 読了」、子ども向けは「よみたい / よんでる / よみおわった」。
- 「記録」「読書記録」「どくしょきろく」「きょうの記録をつける」が混在。
- 「メモ」「ひとことかんそう」など、同じ入力目的に複数ラベルがあり、利用者が迷う可能性。

### 改善提案
- 文言ガイドライン（親向け辞書/子向け辞書）を作り、画面単位ではなくアプリ全体で管理する。

## 子供モードの漢字の使い方
### 評価
- ひらがな寄せはできているが、フォームラベルや一部見出しに漢字が混在。
- 子どもの読字レベル差を考えると、モード切替だけでなく**年齢帯別UI（未就学/低学年）**の余地がある。

### 改善提案
- 子どもモードでは原則ひらがな、必要語のみ漢字＋ルビ（または言い換え）に統一。
- 難語（著者・任意など）は補足ツールチップや言い換えを検討。

## ユーザー導線
### 良い点
- 保護者導線: ダッシュボード→子ども詳細→記録詳細→コメント/リアクション は自然。
- 子ども導線: ホーム→記録登録のFABが常時表示で行動に移しやすい。

### 気になる点
- 初回利用時に「何から始めるか」の案内が薄い（特に保護者の初期セットアップ後）。
- こどもログインで「子どもID」が必須になるケースは、実運用で入力ミスや離脱要因になりやすい。

## スマホ表示を前提としたレイアウトの適切さ
### 良い点
- `max-w-md` / `max-w-xl`中心でモバイル幅に収まりやすい設計。
- 横スクロール（ジャンルタブ・おすすめ本）はコンテンツ特性に合っている。

### 気になる点
- 親向け画面ではアクションボタン群が縦方向に伸び、ファーストビュー圧迫が発生。
- 固定FABとOS UI/ブラウザUIが重なる端末で、下部要素と競合する可能性。

## 記録やコメント等の編集可否
### 現状評価
- 記録は更新可能（ステータス/メモ/読了日）。
- コメントは投稿可能だが、編集・削除導線が見当たらない。
- 記録自体の削除導線も見当たらない。

### ユーザー影響
- 入力ミス修正や誤投稿取り消しができず、長期利用時のストレス要因になる。

### 改善提案
- 最低限「自分のコメント削除」「記録削除（確認ダイアログ）」を追加。
- 監査/履歴要件がある場合は論理削除＋復元期間を検討。

## その他、実利用で気になりそうな点
- 空状態（データ0件）で次アクション提案はあるが、初回オンボーディングとしては弱め。
- エラーメッセージ文言は簡潔だが、再試行方法の具体提示が少ない。
- 検索・スキャン失敗時のフォールバックはあるものの、子どもには原因が伝わりにくい。

## 優先度付き改善アクション（推奨）
1. 子どもモード文言の漢字/かなポリシーを統一し、全画面適用。
2. コメント/記録の編集・削除導線を追加。
3. 親ダッシュボードの上部アクションを整理（優先度順・折りたたみ導入）。
4. 初回利用オンボーディング（3ステップ程度）を追加。
5. 文字サイズ10px帯の見直し（最小12px目安）。
