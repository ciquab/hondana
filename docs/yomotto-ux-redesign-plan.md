# よもっと！(Yomotto!) UI/UX リデザイン計画（実装前プラン）

> 対象: 現行「ほんだな」
> 
> 前提: アプリ名を **「よもっと！」（英語名: Yomotto!）** へ変更
> 
> 制約: 機能・業務ロジック・DB/API・認証モデル・ロール（子ども/保護者）は変更しない

---

## 1. UX problems analysis（現状課題の診断）

仕様と利用文脈（日本の親子、スマホ中心）を前提に、現行UIで起こりやすい課題を整理します。

### 1-1. 画面構造・導線
- 機能は豊富だが、子どもにとって「次に何をすればよいか」が即時に分かりにくい。
- 子ども向け体験と保護者向け体験が混在すると、情報密度が過剰になりやすい。
- 初回利用時に「記録する→達成を感じる」までの導線が長く感じられる可能性。

### 1-2. 情報設計
- 同じ情報（読書状況・反応・メッセージ）が複数画面に分散し、文脈が切れる。
- カレンダー・本棚・タイムラインの関係が見えづらいと、利用価値が直感的に伝わりにくい。

### 1-3. コンポーネント一貫性
- カード・ボタン・バッジ・タグのサイズ/余白/優先度ルールが揃っていないと、可読性が落ちる。
- 子ども向けの柔らかさと保護者向けの信頼感を同じUI原則で統一できていない可能性。

### 1-4. モバイル操作性
- 44px未満のタップ領域、狭い余白、密な情報配置が誤タップを誘発。
- スマホでの片手操作において、重要操作が上部に集中すると使いづらい。

### 1-5. 感情設計
- 記録アプリとして機能していても、
  - 子どもにとっての「できた！」
  - 保護者にとっての「見守れている安心感」
  が日常的に得られないと、継続率が下がる。

## 2. New UX concept（新UXコンセプト）

### コンセプト名
**「まいにち育つ、家族の本だな」**

### デザイン哲学（3原則）
1. **Calm Reading Space（落ち着いた読書空間）**
   - 余白を活かした静かなUIで、読書行為そのものを主役にする。
2. **Visible Growth（成長の見える化）**
   - 冊数・日数・継続・感想が自然に積み上がる体験を中心にする。
3. **Gentle Encouragement（やさしい励まし）**
   - 反応やコメントを「評価」ではなく「応援」として届ける。

### トーン
- 温かいが幼すぎない
- 清潔で現代的
- 子どもは迷わず、保護者は安心できる

## 3. Information architecture（情報設計）

子どもUXと保護者UXを同一プロダクト内で明確に分離し、
「目的単位」で画面責務を再定義します。

## 3-1. Child IA（子ども向け）

### メイン画面
1. **ホーム（きょうのよもっと！）**
   - 今日やること（記録、続きから読む、スタンプ確認）
   - 最新メッセージ
   - ミニ達成表示（連続日数、今月冊数）
2. **きろく（追加・編集）**
   - バーコード登録
   - ステータス選択（読みたい/読んでる/よみおわり）
   - 絵文字スタンプ・タグ入力
3. **ほんだな（本棚）**
   - ステータス別棚（読みたい/読んでる/読了）
   - 書影中心の視覚表示
4. **カレンダー**
   - 日次の読書記録可視化
   - 日付タップで当日の記録一覧
5. **バッジ・メッセージ**
   - 獲得バッジ
   - 保護者メッセージ履歴

### 子ども導線
- ホーム →（主要CTA）→ きろく
- きろく完了 → 成長フィードバック → ホーム
- ホームから本棚/カレンダーへ横展開

## 3-2. Parent IA（保護者向け）

### メイン画面
1. **ダッシュボード**
   - 読書サマリー（今週/今月）
   - 子ども切替
   - 注目トピック（最近の記録、未返信コメント）
2. **タイムライン**
   - 時系列で読書記録と反応履歴を確認
   - コメント・リアクション入力
3. **分析（ジャンル統計）**
   - ジャンル比率、読書傾向
   - 期間切替
4. **子ども別本棚ビュー**
   - 子どもの本棚を保護者視点で閲覧
5. **設定/家族管理**
   - 既存のアカウント・ロール範囲での設定導線

### 保護者導線
- ダッシュボード → タイムライン（反応）
- ダッシュボード → 分析（傾向把握）
- 子ども切替は全主要画面上部で一貫提供

## 4. Navigation design（ナビゲーション設計）

## 4-1. 子どもナビゲーション
- **下部タブ（4〜5項目）**
  - ホーム / きろく / ほんだな / カレンダー / バッジ
- 中央の「きろく」を最も目立つCTAとして配置。
- 片手操作を考慮し、主要操作は下部に集中。

**理由**
- 子どもは深い階層探索より「見える選択肢」を好む。
- 毎日繰り返す行動（記録）を最短1タップ導線にできる。

## 4-2. 保護者ナビゲーション
- **下部タブ + 画面上部セグメント（必要時）**
  - ダッシュボード / タイムライン / 分析 / 本棚
- 上部に常時「子ども切替ドロップダウン」を表示。

**理由**
- 保護者は比較・把握・入力の往復が多いため、並列遷移が有効。
- 子ども切替を固定化することで迷いを減らす。

## 5. Component system（コンポーネント体系）

再利用可能なUI部品を「役割」で定義し、一貫性を担保します。

### 共通コンポーネント
1. **App Header**
   - 画面タイトル、子ども切替（保護者のみ）、補助アクション
2. **Primary Button / Secondary Button / Tertiary Text Button**
3. **Status Chip**
   - 読みたい / 読んでる / よみおわり
4. **Tag Chip**
   - 感情・ジャンルタグ
5. **Reaction Bar**
   - 絵文字スタンプのクイック入力
6. **Empty State Block**
   - 初回導線 + やさしい説明

### 子ども向け主要コンポーネント
1. **Book Shelf Card**
   - 書影、タイトル、進捗ステータス、最終更新
2. **Record Composer Card**
   - バーコード登録入口、状態選択、感情入力
3. **Achievement Banner**
   - 「○日連続！」などの即時達成表示
4. **Badge Tile**
   - バッジ名、達成条件、達成状態
5. **Parent Message Card**
   - 保護者コメント + 応援リアクション

### 保護者向け主要コンポーネント
1. **Dashboard Summary Card**
   - 今週冊数、読了率、連続記録日数
2. **Timeline Record Card**
   - 子ども名、書籍、日時、ステータス、子どもの反応
3. **Comment Composer**
   - 定型リアクション + 自由コメント
4. **Genre Stats Card**
   - 比率表示（円/棒）、期間ラベル
5. **Child Switcher**
   - アバター + 名前 + 学年補助情報

## 6. Design system（デザインシステム）

## 6-1. Color palette（落ち着き + 温かさ）

### Brand
- **Yomotto Blue** `#4A6FA5`（信頼・読書の落ち着き）
- **Yomotto Coral** `#F28C82`（やさしい励まし）

### Neutrals
- `#F8FAFC` 背景
- `#FFFFFF` サーフェス
- `#E5E7EB` 境界
- `#475569` 本文補助
- `#1E293B` 本文主

### Semantic
- Success `#22C55E`
- Warning `#F59E0B`
- Info `#3B82F6`

> 彩度は抑えめ。子ども向けでも背景は白/淡色中心にして可読性優先。

## 6-2. Typography（日本語可読性重視）
- フォント: `Noto Sans JP`（既存環境に合わせ代替可）
- スケール（スマホ基準）
  - Display: 28 / 700
  - H1: 24 / 700
  - H2: 20 / 700
  - H3: 18 / 600
  - Body L: 16 / 500
  - Body M: 14 / 500
  - Caption: 12 / 500
- 行間: 1.5前後（本文）、見出しは1.3

## 6-3. Spacing & layout
- **8pxグリッド**
- 画面左右余白: 16px
- セクション間: 24px
- カード内余白: 12〜16px
- 最小タップ領域: **44px**

## 6-4. Radius & shadow
- Radius: 8 / 12 / 16（カードは12基準）
- Shadow: 
  - Card: `0 2 8 rgba(15,23,42,0.06)`
  - Floating CTA: `0 6 20 rgba(15,23,42,0.12)`
- 境界線を薄く使い、影は控えめ

## 6-5. Interaction rules
- 押下フィードバック: 120ms
- 画面遷移: 180〜220ms、自然なease
- 記録完了時は「小さな達成アニメーション」を付与（過剰演出はしない）

## 7. Redesigned child screens（子ども向け画面再設計）

## 7-1. ホーム
- 上部: 「こんにちは、◯◯さん」+ 今日の一言
- 中央: 今日の主行動カード（記録する）
- 下部: 最近読んだ本、保護者メッセージ、ミニバッジ進捗

## 7-2. きろく（追加/編集）
- ステップ式
  1) バーコードで本を追加
  2) 読書ステータス選択
  3) 気持ちスタンプ/タグ入力
  4) 完了
- 各ステップは1画面1目的で迷わせない

## 7-3. ほんだな
- タブ: 読みたい / 読んでる / よみおわり
- 書影2カラム（スマホ）で一覧性を高める
- タップで詳細（既存機能範囲）

## 7-4. カレンダー
- 月表示 + 日別達成ドット
- 日付タップで記録カードを下部シート表示

## 7-5. バッジ・メッセージ
- バッジを進捗付きで表示（次の目標を明確化）
- メッセージは時系列カードで「応援履歴」として可視化

---

## 8. Redesigned parent screens（保護者向け画面再設計）

## 8-1. ダッシュボード
- ファーストビューに「今週の読書サマリー」
- 主要KPI（冊数、読了、連続日数）を3カードで横並び
- 下部に「最近の記録」ショートリスト

## 8-2. タイムライン
- 子どもの記録を時系列表示
- 各カードにクイックリアクション（絵文字）とコメントボタン
- コメント入力は下部シートで完結

## 8-3. 分析（ジャンル統計）
- 期間フィルタ（今月/3か月/半年）
- ジャンル比率をカード化
- 変化トレンドを簡潔な注釈で補助

## 8-4. 子ども別本棚ビュー
- 子ども切替後、同一構造で本棚表示
- 保護者視点でも「どこまで読んだか」を即把握可能に

既存機能を壊さず、将来機能を追加しやすい情報構造を確保します。
