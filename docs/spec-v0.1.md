# 家族向け読書記録アプリ「ほんだな」仕様書 たたき台

バージョン: 0.1（たたき台）
作成日: 2026年3月3日
想定利用者: 小学生（子ども）＋ 保護者（親）

## 1. プロダクト概要

### 1.1 アプリ名（仮）

**ほんだな** ― 子どもの「読んだ本」「読みたい本」が並ぶ、家族の本棚。

### 1.2 ミッション

小学生が「読んだ喜び」を記録・共有し、親がそれを見守り応援できる。
AIと書籍データベースで、次の「出会う一冊」を自然に提案する。

### 1.3 対象ユーザー

| ロール | 説明 |
|---|---|
| 子どもユーザー | 小学生（低〜高学年を想定）。複数人登録可 |
| 親ユーザー | 保護者。子どものアカウントを管理・閲覧 |

### 1.4 プラットフォーム

- 当面の方針: Webアプリ（レスポンシブ対応）としてスタートし、需要に応じてモバイルアプリ化を検討
- バーコード読み取りはスマホブラウザのカメラAPI（WebRTC / BarcodeDetector API）を使用。子ども・親どちらのデバイスでも動作

## 2. ユーザーストーリー

### 子どもユーザー

- 本のバーコードをスキャンして、かんたんに読書記録をつけたい
- 感想を「えもじ」や選択式でかんたんに入力したい
- 読んだ本が貯まっていくのが嬉しい（達成感・コレクション感）
- 次に読む本を教えてもらいたい
- 難しい本のあらすじをわかりやすく教えてほしい

### 親ユーザー

- 子どもが何を読んでいるか把握したい
- 子どもの感想に「いいね」やコメントをして、読書を応援したい
- 子どもに読ませたい本を「おすすめ」として登録したい
- 複数の子どもをまとめて管理したい
- 子どもの読書傾向を把握し、次の一冊を一緒に探したい

## 3. 機能一覧

### 3.1 アカウント・ファミリー管理

| 機能 | 説明 |
|---|---|
| ファミリー登録 | 親が「ファミリー」を作成し、招待コード or QRコードで子どもアカウントを紐付け |
| マルチ子どもアカウント | 1ファミリーに複数の子どもを登録可能 |
| ロール切り替え | 親アカウントで子どもの画面をプレビュー可能 |
| 子どもログイン | PIN（4桁）またはイラスト選択式ログイン（文字入力が難しい低学年向け） |

### 3.2 読書記録機能（子ども向け）

#### 3.2.1 本の登録

- **バーコードスキャン登録（メイン）**: スマホカメラでISBNバーコードをスキャン → Google Books API / 国立国会図書館APIで書誌情報を自動取得
- **タイトル検索登録**: キーワード検索で本を探して登録
- **手動登録**: データが見つからない場合はタイトル・著者を手入力

#### 3.2.2 読書状態管理

| ステータス | 説明 |
|---|---|
| 読みたい📚 | 「よみたいほん」リスト |
| 読書中📖 | 現在読んでいる本 |
| 読み終わった⭐ | 読書記録の完成 |

#### 3.2.3 読書記録入力（子ども向けに最適化）

- **スタンプ評価**: ★1〜5ではなく「めちゃよかった🌟 / よかった😊 / ふつう😐 / むずかしかった😓」などアイコン選択
- **気持ちタグ**: 「ドキドキした」「笑った」「かなしかった」「ためになった」「また読みたい」などタップ選択
- **ひとことメモ**: 自由テキスト入力（任意）。フリック入力しやすいよう大きめのテキストエリア
- **読んだページ数**: 「最初から最後まで読んだ」「途中まで読んだ」を選択 or ページ数入力
- **読んだ日**: カレンダーから選択（デフォルトは今日）

#### 3.2.4 ゲーミフィケーション（継続モチベーション）

- **読書バッジ**: 「はじめての1冊」「10冊達成」「同じ作家3冊」など達成バッジ
- **読書カレンダー**: 読んだ日に本のアイコンが付く（継続が可視化される）
- **本棚ビジュアル**: 登録した本が棚に並ぶ画面（本の表紙がコレクションになる）
- **読書ポイント**: 記録するとポイントが貯まり、アバターや本棚の見た目をカスタマイズ可能（課金なし）

### 3.3 親向け機能

#### 3.3.1 子どもの読書記録閲覧

- 子ども別ダッシュボード（読んだ冊数・最近読んだ本・読書傾向グラフ）
- 子どもの本棚・記録一覧を閲覧
- 複数の子どもを切り替えて確認

#### 3.3.2 リアクション機能

- **いいね**: 子どもの記録に「いいね」（❤️ / 👍 / 🌟 など複数種類）
- **コメント**: テキストコメントを投稿。子どもの画面に「おや（またはニックネーム）からのメッセージ」として表示
- **子どもへの通知**: 親がコメント・いいねをすると子どもに通知（アプリ内通知）

#### 3.3.3 親からのおすすめ機能

- 親が本を検索して「〇〇へのおすすめ」として登録
- 子どもの「よみたいほん」リストに「ママ/パパからのおすすめ📬」として表示

#### 3.3.4 読書レポート

- 週次 / 月次の読書まとめを自動生成（読んだ冊数・ジャンル傾向・コメント数など）
- 子どもの成長記録として振り返れる（将来的には年次レポートPDFエクスポート）

### 3.4 本の探索・検索機能

#### 3.4.1 本の検索・あらすじ説明

- キーワード検索 → Google Books APIで候補表示
- **AIやさしいあらすじ**: 本のあらすじをAI（Claude API）が小学生向けにやさしく言い換えて表示
  - 難しい漢字にルビを振る（または平仮名混じりで表現）
  - 「どんなお話？」「どんな子におすすめ？」「読んだらどんな気持ちになる？」の3点で構成

#### 3.4.2 次に読む本のレコメンド

**① AIレコメンド（子どもの読書履歴ベース）**
- 読んだ本のジャンル・著者・気持ちタグ・評価を分析
- Claude APIが「あなたにおすすめの次の一冊」をレコメンド（理由付きでわかりやすく提示）
- 「もっとアドベンチャーが読みたい」「怖いのは苦手」などの子どもの好みを会話形式でヒアリングして精度向上（オプション）

**② 外部サービス連携**
- カーリル（図書館API）: 近くの図書館での在庫検索。「近くの図書館にある！」を表示
- 楽天ブックス / Amazon Product Advertising API: 購入リンク表示（親向け）
- 読書メーター / ブクログ的なランキング: 同年代に人気の本を表示（将来検討）

#### 3.4.3 テーマ別本棚

- 「夏休みに読みたい本」「理科が好きな子向け」「ドキドキ冒険もの」などキュレーション棚を管理者が作成
- AIが子どもの読書傾向に合ったテーマ棚を自動提案

### 3.5 AIチャット機能（「本のともだち」）

子どもが本について気軽に質問できるAIアシスタント機能。

- 「この本のここがわからなかった」→ AI が解説
- 「〇〇みたいな本を教えて」→ レコメンド
- 「この本ってどういうこと？」→ テーマ・背景を子ども向けに解説
- 保護者が会話ログを確認可能（安全性担保）
- 不適切なコンテンツフィルター必須

## 4. 外部API・サービス連携

| サービス | 用途 | 備考 |
|---|---|---|
| Google Books API | 書誌情報取得（ISBN検索・タイトル検索） | 無料枠あり |
| 国立国会図書館 書誌情報API | 日本語書籍の補完 | 無料 |
| Claude API（Anthropic） | AIあらすじ解説・レコメンド・チャット | 有料 |
| カーリルAPI | 図書館在庫検索 | 無料（利用登録要） |
| 楽天ブックスAPI / Amazon PA API | 購入リンク・書影補完 | 要申請 |

## 6. 技術スタック（候補）

Claude Code・Codexでのコード生成を前提とした構成案。

### フロントエンド

- React（Next.js） + TypeScript
- スタイリング: Tailwind CSS
- バーコードスキャン: @zxing/library または BarcodeDetector API（ブラウザネイティブ）
- 状態管理: Zustand or React Query

### バックエンド

- Next.js API Routes or Hono（エッジランタイム対応）
- 認証: NextAuth.js（ファミリーアカウント管理）

### データベース

- Supabase（PostgreSQL + リアルタイム更新 + Storage）
  - 本の表紙画像キャッシュ
  - 読書記録・コメントのリアルタイム反映

### AI

- Claude API（Anthropic）: あらすじ解説・レコメンド・チャット
- モデル: claude-sonnet-4-6（コスト・品質バランス）

### ホスティング

- Vercel（フロントエンド + API Routes）
- Supabase（DB・ストレージ）

## 7. 非機能要件

| 項目 | 要件 |
|---|---|
| セキュリティ | 子どものデータは家族外に非公開。AIチャットに不適切語フィルター |
| COPPA / 児童プライバシー | 13歳未満のデータ収集は親の同意を前提とした設計 |
| パフォーマンス | バーコードスキャン → 書誌情報取得 3秒以内 |
| オフライン対応 | 記録入力はオフラインでも可能（後でSync） |
| アクセシビリティ | 文字サイズ大きめ・ルビ対応・タップ領域を広く |
| データポータビリティ | 読書記録のCSV / JSONエクスポート機能 |

## 8. フェーズ計画

- ファミリーアカウント作成・子どもアカウント登録
- バーコードスキャン / 書誌情報取得 / 読書記録登録
- 親による閲覧・いいね・コメント
