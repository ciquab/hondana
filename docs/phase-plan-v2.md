## 再構築フェーズ計画

### Step 2: Phase 1 完成 ― 書籍データ連携 & UX 強化

**目標**: 手動入力のみの MVP を「本を簡単に登録できる」状態にし、親子で使える UI に仕上げる。

| # | 機能 | 概要 |
|---|---|---|
| 2-1 | バーコードスキャン | @zxing/library or BarcodeDetector API でISBN読み取り |
| 2-2 | Google Books API 連携 | ISBN / タイトル検索 → 書誌情報自動取得（タイトル・著者・表紙画像） |
| 2-3 | 国立国会図書館 API 連携 | Google Books で見つからない日本語書籍のフォールバック |
| 2-4 | 本棚ビジュアル | 表紙画像を使った本棚風表示（コレクション感） |
| 2-5 | いいねリアクション | 記録への ❤️ / 👍 / 🌟 リアクション（DB + UI） |
| 2-6 | レスポンシブ UI 改善 | モバイルファースト調整・タップ領域拡大・フォント最適化 |

**成果物**: 親のスマホで子どもの本をスキャン→登録→閲覧→リアクションの一連フローが完成

---

### Step 3: 子ども向け体験 ― 子ども UI & ゲーミフィケーション

**目標**: 子どもが自分で使える UI と、継続モチベーションの仕組みを構築。

| # | 機能 | 概要 |
|---|---|---|
| 3-1 | 子ども PIN ログイン | 4桁 PIN / イラスト選択式の簡易認証 |
| 3-2 | スタンプ評価 | 🌟😊😐😓 アイコン選択式の感想入力 |
| 3-3 | 気持ちタグ | 「ドキドキした」「笑った」などタップ選択タグ |
| 3-4 | 読書バッジ | 「はじめての1冊」「10冊達成」などの達成バッジ |
| 3-5 | 読書カレンダー | 読んだ日にアイコンが付くカレンダー表示 |
| 3-6 | ファミリー招待 | 招待コード / QRコードで家族メンバー追加 |
| 3-7 | おやからのメッセージ画面 | コメント・いいねを子ども向けに表示 |

**成果物**: 子どもが自分の端末で PIN ログイン→記録→バッジ獲得→親のメッセージ確認

詳細設計提案: [Step 3 詳細設計提案](./step3-detailed-design.md)

---

### Step 4: AI 機能 ― あらすじ解説 & レコメンド（仕様書 Phase 2）

**目標**: Claude API を活用し「次の一冊との出会い」を支援。

| # | 機能 | 概要 |
|---|---|---|
| 4-1 | AI やさしいあらすじ | 本の詳細画面で Claude API による子ども向けあらすじ生成 |
| 4-2 | AI レコメンド | 読書履歴ベースで「おすすめの次の一冊」を提案 |
| 4-3 | カーリル API 連携 | 近くの図書館での在庫検索・表示 |
| 4-4 | 読書ポイント | 記録でポイント獲得 → 本棚カスタマイズに使用 |
| 4-5 | AI コスト管理 | リクエスト数上限・キャッシュ戦略 |

**成果物**: 本の詳細に「やさしいあらすじ」表示、ホーム画面に「おすすめの一冊」カード

---

### Step 5: コミュニケーション強化（仕様書 Phase 3）

**目標**: 家族間のコミュニケーションを深め、読書習慣の定着を支援。

| # | 機能 | 概要 |
|---|---|---|
| 5-1 | AI チャット「本のともだち」 | 子どもが本について質問できるアシスタント |
| 5-2 | 親からのおすすめ送信 | 親が本を検索 → 子どもの「よみたいほん」に追加 |
| 5-3 | 読書レポート（月次） | 読んだ冊数・ジャンル傾向・コメント数の自動集計 |
| 5-4 | アプリ内通知 | 親のリアクション → 子どもへ通知 |
| 5-5 | テーマ別本棚 | キュレーション棚（管理者作成 + AI 提案） |
| 5-6 | 会話ログ閲覧（親） | AI チャットの安全性担保 |

**成果物**: AI チャット・おすすめ機能・レポートが揃い、家族の読書サイクルが回る状態

---

### Step 6: プロダクション & 拡張（仕様書 Phase 4）

**目標**: 本番運用品質に引き上げ、将来拡張の基盤を整備。

| # | 機能 | 概要 |
|---|---|---|
| 6-1 | E2E テスト | Playwright による主要フローの自動テスト |
| 6-2 | データエクスポート | 読書記録の CSV / JSON エクスポート |
| 6-3 | 年次レポート PDF | 1年間の読書まとめを PDF 出力 |
| 6-4 | PWA 対応 | オフライン記録入力 → 復帰時 Sync |
| 6-5 | モバイルアプリ検討 | React Native / Expo への移行判断 |
| 6-6 | 読み聞かせ記録 | 親が読んだ本の記録対応 |
| 6-7 | 外部購入リンク | 楽天ブックス / Amazon リンク表示（親向け） |
