# Phase 0 ブランド表記統一ルール（よもっと！ / Yomotto!）

## 目的
UI/UX移行中にブランド表記が混在しないよう、文言ルールを先に固定する。

---

## 1. 正式表記

- 日本語正式名: **よもっと！**
- 英語正式名: **Yomotto!**

> 原則として日本語UIは「よもっと！」を使用する。

---

## 2. 文脈別の使用ルール

| 文脈 | 表記ルール | 例 |
|---|---|---|
| アプリ内の画面タイトル | よもっと！ | `よもっと！ホーム` |
| ログイン/導入文言 | よもっと！ | `よもっと！へようこそ` |
| 技術文書/英語併記が必要 | よもっと！(Yomotto!) | `よもっと！(Yomotto!) UX方針` |
| URL/slug/識別子 | 既存仕様維持（必要時のみ） | 既存ルートを原則維持 |

---

## 3. 置換対象（Phase 1で優先対応）

1. ヘッダー/ナビゲーション表示名
2. ページタイトル・空状態メッセージ
3. ログイン・招待・設定の案内文
4. ドキュメント内の対外表記

---

## 4. 禁止事項

- 「ほんだな」「Hondana」を新規追加しない。
- `よもっと`（感嘆符なし）をUIの正式名として使わない。
- `Yomotto！`（全角感嘆符）の混在を許容しない。

---

## 5. QAチェック項目

- [x] 画面タイトルに旧名称が残っていない
- [x] 空状態・エラーメッセージに旧名称が残っていない
- [x] メタ情報・ヘルプ文言の表記が統一されている
- [x] 日本語UIで英語名を過剰表示していない
