# Phase 0 UX監査シート（よもっと！ / Yomotto!）

## 目的
- UI/UX移行前に、現行画面と導線を可視化し、Phase 1以降の改修優先度を確定する。
- 対象は表示・導線・一貫性。機能・業務ロジック・DB/API・認証・ロールは変更しない。

---

## 3. 現状主要導線（As-Is）

### 子ども
1. `/kids/home` に到達
2. `/kids/records` へ遷移し記録入力
3. 完了後にホームへ戻る/他画面へ遷移
4. `/kids/messages`・`/kids/badges` で応援/達成を確認

### 保護者
1. `/dashboard` で全体把握
2. `/children/[childId]` または `/records/[recordId]` で詳細閲覧
3. コメント/リアクション実行
4. 設定系で家族情報の管理

---

## 4. UXリスク評価（Phase 0判定）

| リスク | 影響 | 優先度 | 解決フェーズ |
|---|---|---|---|
| 子ども記録までの導線が冗長化 | 記録完了率低下 | 高 | Phase 2 |
| 保護者の反応導線が分散 | コメント率低下 | 高 | Phase 3 |
| コンポーネント見た目不整合 | 信頼感低下 | 高 | Phase 1 |
| 表記ゆれ（ほんだな/よもっと！） | ブランド認知低下 | 高 | Phase 1〜4 |
| 小画面で情報密度過多 | 誤タップ増加 | 高 | Phase 1/4 |
