# デザインAI向けプロンプト案（ほんだな）

このドキュメントは、現状の実装を踏まえてデザインAIに渡すためのプロンプト案です。
「親モード」と「こどもモード」を同時に扱う前提で、MVPの画面構造に沿ってモックを作れるようにしています。

## 1. 前提共有（デザインAIに最初に渡す要件）

```text
あなたはプロダクトデザイナーです。家族向け読書記録アプリ「ほんだな」のUIモックを作成してください。

# プロダクト概要
- 日本の家庭向けアプリ
- 親が子どもの読書記録を管理し、子ども自身も「こどもモード」で記録できる
- 主な価値: 読書習慣の可視化、家族のコミュニケーション、子どもの達成感

# ユーザー
- 親: 20-40代、スマホ中心、短時間で状況把握したい
- 子ども: 6-12歳、漢字よりひらがな中心、視覚的で直感的な操作が必要

# 既存情報設計（主要画面）
親モード:
1) ログイン / サインアップ
2) ダッシュボード（子ども一覧、家族設定、子ども追加、招待、こどもモード導線）
3) 子ども別の読書記録一覧（読みたい / 読んでる / 読了）
4) 記録詳細（コメント・リアクション）
5) 家族設定（招待コード発行、子どもログインURL/QR）
6) 子ども作成
7) 招待コード参加

こどもモード:
1) こどもログイン（4桁PIN）
2) こどもホーム（記録作成・本だな・カレンダー・メッセージ）
3) 記録作成（バーコード、書名検索、感情スタンプ）
4) 本だな（表紙中心の棚ビュー）
5) メッセージ一覧

# デザイン方針
- 親モード: 信頼感、見やすさ、情報整理、落ち着いたトーン
- こどもモード: 明るく楽しい、達成感が出る、誤操作しにくい
- 同一プロダクトとしての統一感は維持（タイポ・余白・コンポーネント規則）

# 制約
- モバイルファースト（360-390px幅）＋タブレット/PCにも自然に拡張
- 日本語UI
- アクセシビリティ重視（文字サイズ、コントラスト、タップ領域）
- 実装しやすい現実的なUI（過度に複雑なアニメーションは不要）

# 出力形式
- まず「ビジュアルコンセプト」を2案提示
- 次に主要画面のモック（親モード/こどもモード）
- 最後にカラー・タイポ・主要コンポーネント仕様をテキストで整理
```

## 2. 方向性を深掘りする追加プロンプト

### A. コンセプト比較をしたいとき

```text
以下の2方向で比較してください。

A案: 絵本の温かさ
- 紙の質感や柔らかい角丸
- 暖色ベース、安心感
- 家族のぬくもりを感じる

B案: すっきり学習アプリ
- 余白を広く、情報優先
- 高い可読性
- 習慣化ダッシュボードが強い

それぞれについて、
1) トーン&マナー
2) 推奨カラーパレット
3) 親モード/こどもモードへの適用方法
4) 向いているユーザー心理
を簡潔に説明してください。
```

### B. こどもモードだけ強化したいとき

```text
こどもモードの体験を最優先で磨いてください。

重視ポイント:
- ひらがな中心の読みやすい文言
- 1画面1目的（迷わない導線）
- 達成演出（バッジ、連続記録、読了時のごほうび感）
- 記録作成で「バーコード」「検索」「手入力」の3手段を迷わず選べる

必要画面:
1) こどもログイン
2) こどもホーム
3) 記録作成
4) 本だな
5) メッセージ

各画面で、主ボタン・副ボタン・注意喚起・空状態の見せ方を具体化してください。
```

## 3. 最終成果物を実用レベルに寄せるプロンプト

```text
このモックはエンジニア実装に引き継ぎます。以下を必ず出力してください。

1) 画面ごとのレイアウト仕様
- ヘッダー構成
- セクション順序
- CTA配置
- カード/リストのルール

2) デザイントークン
- 色（primary / secondary / accent / success / warning / error / neutral）
- タイポ（見出し、本文、補助）
- 角丸、影、余白スケール

3) コンポーネント一覧
- ボタン（親モード用、こどもモード用）
- 入力フォーム
- 本カード
- ステータスチップ（読みたい/読んでる/読了）
- バッジ表示
- 空状態カード

4) レスポンシブ指針
- モバイル -> タブレット -> デスクトップで何をどう変えるか

5) アクセシビリティ
- 最小文字サイズ
- コントラスト基準
- フォーカス表示
- タッチターゲット最小サイズ

最後に「実装優先順位（P1/P2/P3）」を付けてください。
```

## 4. 実際に使うときのコツ

- 最初は「前提共有」+「A.コンセプト比較」だけ投げる
- 良い方の案を選んでから「3.最終成果物」を追加する
- 1回で完璧を狙わず、
  - 画面構成
  - ビジュアル
  - コンポーネント仕様
    の順で段階的に詰める
