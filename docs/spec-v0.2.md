# 家族向け読書記録アプリ「ほんだな」仕様書

バージョン: 0.2
作成日: 2026年3月3日
更新日: 2026年3月7日（実装状況に合わせた改訂）
想定利用者: 小学生（子ども）＋ 保護者（親）

> **凡例**: ✅ 実装済み ／ 🔄 一部実装 ／ 📋 未実装（計画あり）

---

## 1. プロダクト概要

### 1.1 アプリ名

**ほんだな** ― 子どもの「読んだ本」「読みたい本」が並ぶ、家族の本棚。

### 1.2 ミッション

小学生が「読んだ喜び」を記録・共有し、親がそれを見守り応援できる。
AIと書籍データベースで、次の「出会う一冊」を自然に提案する。

### 1.3 対象ユーザー

| ロール | 説明 |
|---|---|
| 子どもユーザー | 小学生（低〜高学年を想定）。複数人登録可 |
| 親ユーザー | 保護者。子どものアカウントを管理・閲覧 |

### 1.4 プラットフォーム

- Webアプリ（レスポンシブ対応）としてスタート、需要に応じてモバイルアプリ化を検討 ✅
- バーコード読み取りはスマホブラウザのカメラAPI（BarcodeDetector API / @zxing/library フォールバック）✅

---

## 2. ユーザーストーリー

### 子どもユーザー

- 本のバーコードをスキャンして、かんたんに読書記録をつけたい ✅
- 感想を「えもじ」や選択式でかんたんに入力したい ✅
- 読んだ本が貯まっていくのが嬉しい（達成感・コレクション感）✅
- 次に読む本を教えてもらいたい 📋（Step 4）
- 難しい本のあらすじをわかりやすく教えてほしい 📋（Step 4）

### 親ユーザー

- 子どもが何を読んでいるか把握したい ✅
- 子どもの感想に「いいね」やコメントをして、読書を応援したい ✅
- 子どもに読ませたい本を「おすすめ」として登録したい 📋（Step 5）
- 複数の子どもをまとめて管理したい ✅
- 子どもの読書傾向を把握し、次の一冊を一緒に探したい 🔄（ジャンル集計のみ実装済み）

## 3. 機能一覧

### 3.1 アカウント・ファミリー管理

| 機能 | 説明 | 状態 |
|---|---|---|
| ファミリー登録 | 親が「ファミリー」を作成し、招待コード or QRコードで子どもアカウントを紐付け | ✅ |
| マルチ子どもアカウント | 1ファミリーに複数の子どもを登録可能 | ✅ |
| ロール切り替え | 親アカウントで子どもの画面をプレビュー可能 | 📋 |
| 子どもログイン（PIN） | 4桁 PIN による簡易認証。ロックアウト・監査ログ付き | ✅ |
| 子どもログイン（イラスト選択） | 文字入力が難しい低学年向けのイラスト選択式 | 📋 |

### 3.2 読書記録機能（子ども向け）

#### 3.2.1 本の登録 ✅

- **バーコードスキャン登録（メイン）**: スマホカメラでISBNバーコードをスキャン → Google Books API / 国立国会図書館API / OpenBD で書誌情報を自動取得
- **タイトル検索登録**: キーワード検索で本を探して登録
- **手動登録**: データが見つからない場合はタイトル・著者を手入力

#### 3.2.2 読書状態管理 ✅

| ステータス | 説明 |
|---|---|
| 読みたい📚 | 「よみたいほん」リスト |
| 読書中📖 | 現在読んでいる本 |
| 読み終わった⭐ | 読書記録の完成 |

#### 3.2.3 読書記録入力（子ども向けに最適化）

| 項目 | 説明 | 状態 |
|---|---|---|
| スタンプ評価 | 「すごくよかった🌟 / たのしかった😊 / ふつう😐 / むずかしかった😓」選択。スタンプ別のハイライト色（amber / emerald / slate / violet） | ✅ |
| 気持ちタグ | 「ドキドキした」「笑った」「かなしかった」「ためになった」「また読みたい」などタップ選択 | ✅ |
| ジャンル選択 | 「物語 / 図鑑・ノンフィクション / マンガ / 詩・絵本 / その他」から選択（子ども・親どちらの記録フォームにも追加） | ✅ |
| ひとことメモ | 自由テキスト入力（任意）。大きめのテキストエリア | ✅ |
| 読んだページ数 | 「最初から最後まで読んだ / 途中まで読んだ」選択またはページ数入力 | ✅ |
| 読んだ日 | カレンダーから選択（デフォルトは今日） | ✅ |

#### 3.2.4 ゲーミフィケーション（継続モチベーション）

| 機能 | 説明 | 状態 |
|---|---|---|
| 読書バッジ（冊数系） | 「はじめての1冊」「10冊達成」「同じ作家3冊」など達成バッジ。獲得時アニメーション付き | ✅ |
| ジャンル別バッジ | 「はじめて物語を読んだ！」「3ジャンル制覇！」などジャンル横断バッジ | ✅ |
| 読書カレンダー | 読んだ日に本のアイコンが付くカレンダー表示 | ✅ |
| 本棚ビジュアル | 登録した本が棚に並ぶ画面。ジャンル別タブ切り替え対応。棚背景＋棚板の2層構造 | ✅ |
| 読書ポイント | 記録でポイント獲得 → 本棚カスタマイズ（課金なし） | 📋（Step 4） |

### 3.3 親向け機能

#### 3.3.1 子どもの読書記録閲覧

#### 3.4.3 テーマ別本棚

### 3.5 AIチャット機能（「本のともだち」）

子どもが本について気軽に質問できるAIアシスタント機能。 📋（Step 5）

- 「この本のここがわからなかった」→ AI が解説
- 「〇〇みたいな本を教えて」→ レコメンド
- 保護者が会話ログを確認可能（安全性担保）
- 不適切なコンテンツフィルター必須

- Tailwind CSS ✅

- Next.js Server Actions ✅
- Supabase（PostgreSQL + RLS + Storage）✅

- Claude API（Anthropic）: あらすじ解説・レコメンド・チャット 📋（Step 4〜5）

### ホスティング

- Vercel（フロントエンド + Server Actions）
- Supabase（DB・ストレージ）

## 7. セキュリティ

| 項目 | 実装内容 | 状態 |
|---|---|---|
| RLS（Row Level Security） | 家族スコープで全テーブルにポリシー設定。IDOR対策済み | ✅ |
| 子どもPINのロックアウト | 試行回数超過でロック。監査ログ記録 | ✅ |
| セッション管理 | 親: Supabase Auth セッション。子ども: 独自セッショントークン（httpOnly Cookie）| ✅ |
| ファミリー招待監査ログ | 招待の作成・承認・失効を記録 | ✅ |
| 不審アクティビティ検出 | alert_candidates テーブルに記録 | ✅ |
| AIコンテンツフィルター | チャット機能の不適切コンテンツ対策 | 📋（Step 5） |
| COPPA 対応 | 13歳未満データ収集は親の同意を前提とした設計 | 🔄（設計方針のみ） |
