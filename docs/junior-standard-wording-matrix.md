# junior / standard 文言対照表（総点検）

## 統一方針
- **junior**: 読みやすさ優先。和語はひらがな、UI固有語は必要に応じてカタカナ。
- **standard**: 学齢相当の漢字を使う。
- 用語統一:
  - ナビゲーション語は `ホーム` に統一（junior/standard共通）
  - コミュニケーション語は junior=`おてがみ`, standard=`メッセージ`
  - 固有UI語は `カレンダー` / `バッジ` を維持

## 主要対照表
| 画面 | 用途 | junior | standard | 判定 |
|---|---|---|---|---|
| ホーム | ヘッダー | `◯◯ の ホーム` | `◯◯ のホーム` | OK |
| ホーム | メッセージ導線ラベル | `おてがみ` | `メッセージ` | OK |
| ホーム下部カード | 見出し | `おうちのひとからのおてがみ` | `おうちのひとからのメッセージ` | OK |
| メッセージ | 空状態タイトル | `まだ おてがみは ないよ` | `まだメッセージはありません` | OK |
| メッセージ | 空状態説明 | `...おてがみが とどくかも！` | `...メッセージが届くかもしれません。` | **修正済み** |
| 全kids画面 | 戻る導線 | `ホーム` / `ホームへ` | `ホーム` / `ホームに戻る` | OK |
| ホーム | カレンダー導線 | `カレンダー` | `カレンダーを見る` | OK |
| カレンダー | 月送り | `◀ まえ` / `つぎ ▶` | `◀ 前の月` / `次の月 ▶` | OK |
| カレンダー | バッジ見出し | `げっとしたバッジ` | `げっとしたバッジ` | OK |

## 点検で見つかった修正漏れ
1. `app/kids/messages/page.tsx` の junior 空状態説明に `メッセージ` が残っていたため `おてがみ` に修正。

## 今回の結論
- 「全部ひらがな」でも「全部カタカナ」でもなく、**意味役割ごとに統一**するのが最も自然。
- 具体的には `ホーム` / `カレンダー` / `バッジ` はカタカナ維持、`メッセージ` は junior で `おてがみ` に寄せる方針を採用。
