# よもっと！(Yomotto!) UI/UX 詳細移行フェーズ計画

> 参照元: `docs/yomotto-ux-redesign-plan.md`  
> 目的: 既存機能・ロジック・DB/API・認証・ロールを変更せず、UI/UXを段階的に移行する

---

## 0. 移行方針（前提）

- **機能凍結**: 新機能追加は行わず、現行機能の見せ方・導線・一貫性のみ改善。
- **リスク最小化**: 一度に全画面置換せず、フェーズ単位でロールアウト。
- **計測駆動**: 各フェーズで定量/定性の確認を実施し、次フェーズ進行可否を判定。
- **ブランド統一**: UI文言・表示名を「よもっと！(Yomotto!)」へ統一。

---

## 1. フェーズ全体像

| Phase | 期間目安 | 対象 | ゴール | リリース形態 |
|---|---:|---|---|---|
| Phase 0 | 1週間 | 設計基盤 | 現行棚卸し/計測基準/移行準備 | 内部のみ |
| Phase 1 | 1〜2週間 | デザイン基盤 | デザインシステム・共通コンポーネント適用 | 段階公開 |
| Phase 2 | 2週間 | 子ども導線 | 子ども主要導線（ホーム→記録→達成）最適化 | 段階公開 |
| Phase 3 | 2週間 | 保護者導線 | 保護者ダッシュボード・タイムライン再設計 | 段階公開 |
| Phase 4 | 1〜2週間 | 横断最適化 | ナビ/アクセシビリティ/パフォーマンス調整 | 全体公開 |
| Phase 5 | 継続 | 運用改善 | 利用データをもとに微調整・定着化 | 定常運用 |

## 2. Phase 0（現状把握・移行準備）

### 2-1. 実施内容
1. **画面・コンポーネント棚卸し**
   - 子ども/保護者それぞれの画面一覧、UI部品一覧、重複/不整合の可視化。
2. **導線マップ作成**
   - 主要タスク（記録、閲覧、コメント）を現状フローとして整理。
3. **計測指標定義**
   - 主要KPIとイベント計測名を固定。
4. **ブランド反映方針確定**
   - 「ほんだな」表記の置換方針（UI文言、meta、ヘルプ、通知文言）。

### 2-2. 完了条件（DoD）
- 画面一覧・導線マップ・コンポーネント一覧が承認済み。
- KPI計測表（現状値取得可能）が整備済み。
- ブランド表記ガイド（よもっと！/Yomotto!）が合意済み。

### 2-3. 成果物
- UX監査シート
- 計測イベント一覧
- 表記統一ルール

---

## 3. Phase 1（デザインシステム適用）

### 3-1. 実施内容
1. **トークン定義**
   - Color, Typography, Spacing, Radius, Shadow を変数化。
2. **共通コンポーネント刷新**
   - Button, Card, Chip, Header, Empty State, Bottom Nav を新規約へ統一。
3. **モバイル操作基準適用**
   - 44pxタップ領域、8pxグリッド、情報密度基準を全主要画面に適用。
4. **命名変更反映**
   - ヘッダー・タイトル・案内文を「よもっと！」に統一。

### 3-2. 完了条件（DoD）
- 主要共通コンポーネントにデザインシステムが反映。
- 見た目の不整合（色/余白/角丸/影）を基準値で統一。
- 既存機能の挙動差分がない。

### 3-3. リスクと対策
- **リスク**: 小さな見た目変更で意図せずレイアウト崩れ。  
  **対策**: コンポーネント単位のスクリーンショット比較、段階リリース。

---

## 4. Phase 2（子ども向けUX移行）

### 4-1. 優先度
- 最優先: **ホーム → きろく → 完了体験**
- 次点: 本棚 / カレンダー / バッジ・メッセージ

### 4-2. 実施内容
1. **子どもホーム再設計**
   - 今日の行動CTAを明確化（「きろくする」を主導線に固定）。
2. **記録フローの段階UI化**
   - バーコード→ステータス→気持ち入力を1画面1目的で分離。
3. **達成フィードバック改善**
   - 記録完了時の達成表示（連続日数など）を明瞭化。
4. **本棚・カレンダーの情報整理**
   - ステータス別本棚と日次記録の関係を理解しやすく配置。

### 4-3. 完了条件（DoD）
- 子ども主要導線のタスク完了率が改善。
- 誤タップ率・離脱率が基準値より改善。
- 保護者メッセージ到達率（子ども画面での閲覧）が維持/改善。

### 4-4. 計測KPI
- 記録開始率
- 記録完了率
- 記録完了までの平均操作時間
- 子どもホームからの本棚/カレンダー遷移率

---

## 5. Phase 3（保護者向けUX移行）

### 5-1. 優先度
- 最優先: ダッシュボード / タイムライン（コメント導線）
- 次点: 分析 / 子ども別本棚

### 5-2. 実施内容
1. **ダッシュボード再設計**
   - 今週・今月サマリーを先頭に配置し、把握を高速化。
2. **タイムライン操作改善**
   - リアクション/コメントをカード内で簡潔に実行可能に。
3. **子ども切替の一貫化**
   - 主要画面上部へ統一配置。
4. **分析ビュー整理**
   - 期間比較とジャンル傾向を短時間で把握できる構成へ。

### 5-3. 完了条件（DoD）
- 保護者の主要タスク時間（閲覧→反応）が短縮。
- コメント/リアクション実行率が改善。
- 子ども切替時の迷い（誤操作/戻り）が減少。

### 5-4. 計測KPI
- 保護者ログイン後の初回アクションまでの時間
- コメント投稿率
- リアクション実行率
- ダッシュボード経由のタイムライン遷移率

---

## 6. Phase 4（横断最適化・品質保証）

### 6-1. 実施内容
1. **ナビゲーション最終調整**
   - 子ども/保護者双方のBottom Nav視認性と現在地表示を最適化。
2. **アクセシビリティ改善**
   - 文字サイズ、コントラスト、タップ領域、フォーカス順を確認。
3. **パフォーマンス調整**
   - 主要画面の初期表示速度・体感速度を改善。
4. **文言統一の最終監査**
   - 「よもっと！」名称統一漏れをゼロ化。

### 6-2. 完了条件（DoD）
- 重大なUIバグ・導線欠損が解消。
- 主要画面でモバイル利用性基準を満たす。
- ブランド文言の統一完了。

---

## 7. Phase 5（運用最適化・定着）

### 7-1. 実施内容
- 定例でKPIレビュー（週次/隔週）。
- ユーザーフィードバック（親子）に基づく微調整。
- 将来機能（AIおすすめ等）を入れやすいUIスロットの維持確認。

### 7-2. 完了条件（DoD）
- 新UIの利用定着（継続率・再訪率改善）が確認できる。
- 次期改善バックログが優先順位付きで管理される。

---

## 8. リリース戦略（安全な段階移行）

1. **内部検証（開発/QA）**
2. **限定公開（小規模ユーザー）**
3. **段階展開（比率を上げる）**
4. **全体公開**

### ロールバック条件
- 主要KPIが一定閾値以上悪化
- 主要導線（記録/コメント）に致命的障害
- UI崩れで操作不能箇所が複数端末で再現

### ロールバック方式
- 画面単位・ルート単位で旧UIへ切り戻せる構成を維持。
- データ構造・API変更を伴わないため、UIのみ安全に切り戻し可能。

---

## 9. 体制・役割分担（推奨）

- **UX/プロダクト**: 要件優先度・体験品質評価
- **フロントエンド**: デザインシステム実装・画面移行
- **QA**: 端末別検証・回帰確認
- **分析担当**: KPI計測・比較レポート

## 10. チェックリスト（フェーズゲート）

### 各フェーズ共通
- [ ] 機能追加/削除が発生していない
- [ ] DB/API/認証ロジックに変更がない
- [ ] 子ども/保護者どちらかの体験だけが劣化していない
- [ ] 44pxタップ領域・8pxグリッド基準を満たす
- [ ] ブランド名称「よもっと！(Yomotto!)」の統一が維持される

### 最終ゲート
- [ ] 子ども: 記録完了までの体験が短く分かりやすい
- [ ] 保護者: 活動把握と応援アクションが素早い
- [ ] 全体: 一貫した「温かく信頼できる」トーンを実現

---

## 11. 期待効果（移行完了時）

- 子ども: 「迷わず記録できる」「達成が気持ちよく見える」
- 保護者: 「状況把握しやすい」「応援しやすい」
- サービス: 体験一貫性向上により継続利用を促進
