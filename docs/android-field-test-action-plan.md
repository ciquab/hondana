# Android実機テスト指摘への改善計画（追加確認反映版）

## 0. 先に結論（対応順序）
1. **認証UX（大人ログイン）**: ログイン中表示の明確化 + パスワード再設定導線
2. **スマホ表示崩れ修正**: 「〇〇のホーム」「ログアウト」の改行対策
3. **情報設計の不足補完**: 家族名の表示追加、子どもログインURLコピー導線
4. **文言品質改善**: 年齢適応文言（junior/standard）と表記ゆれ統一、親画面の漢字修正
5. **バッジ体験の再設計**: バッジ詳細ページ + 一覧導線 + junior/standard文言分岐
6. **スキャナー体験改善**: バーコード検出条件を明確化（枠外誤検出の抑制）
7. **ビジュアル改善**: こども読書記録カードのトーン統一とノート風UI

## 1. 仕様調査（ひらがな化は年齢切替仕様か？）

### 調査結果
- 年齢表示モードは `AgeMode = 'junior' | 'standard'`。
- 判定ロジックは以下:
  - `age_mode_override` が `junior` / `standard` の場合はそれを優先
  - `auto` の場合は `birth_year` から年齢算出し、**8歳以下なら junior**
  - 生年不正/未設定は `standard` にフォールバック
- 文言切替は `ageText(mode, { junior, standard })` で実装済み。

→ したがって、**「ひらがな化は年齢に応じて切り替える」が現行仕様**。

### 今回の指摘との関係
- 「8歳なのに漢字で表示される」は、仕様がないのではなく、
  1) 該当文言に `ageText` が未適用
  2) `junior` 側の文言が漢字のまま
  3) child の `birth_year`/`age_mode_override` 設定が想定と違う
  のいずれか（または複合）で発生する可能性が高い。

## 2. 追加確認項目（次Sprint前の確認依頼分）

### H. 家族作成時に設定した「家族名」の表示が見当たらない
**確認結果**
- 家族は `families(id, name)` で取得しているが、主要UI（ダッシュボード等）で名称表示していない。
- 家族設定画面でも「家族作成済み」は表示されるが、作成済み家族名そのものは見えづらい。

**対応策**
- ダッシュボード上部に「現在の家族名」表示を追加（複数家族なら選択中家族名を表示）。
- 家族設定画面にも「現在の家族名」セクションを追加し、作成結果の見える化を行う。
- 将来の複数家族対応を見据えて、`currentFamilyId` / `currentFamilyName` をUIで明示。

**受け入れ基準**
- 家族作成後、少なくとも2画面（ダッシュボード/家族設定）で家族名を確認できる。

### I. 子どもログインリンクURLのコピー導線
**確認結果**
- 子どもログインURLは文字列表示のみで、コピーボタンがない。

**対応策**
- 子どもごとのログインURL表示領域に「URLをコピー」ボタンを追加。
- `navigator.clipboard.writeText` 成功時のトースト/文言（例: 「コピーしました」）を表示。
- 失敗時のフォールバック（手動選択の案内）を実装。

**受け入れ基準**
- 1タップでURLコピーでき、成功/失敗がユーザーに分かる。

### J. おやモード個別読書記録ヘッダー「きろく」→「記録」
**確認結果**
- 親画面の記録詳細タイトルが「きろくの詳細」になっている。

**対応策**
- 親画面は漢字トーンに合わせて「記録の詳細」へ修正。
- こども画面のひらがなトーンとは分離（年齢モードの責務と混在させない）。

**受け入れ基準**
- 親向け導線上で「きろく」表記が残らない。

### K. バッジタイトル・説明のjunior/standard分岐 + 詳細導線不足
**確認結果**
- バッジはホーム/カレンダーで一覧表示されるが、タップで詳細へ遷移しない。
- バッジ文言は単一フィールド（name/description）で、年齢モード別の表示分岐がない。

**対応策**
- バッジ詳細ページ（例: `/kids/badges/[badgeId]`）を新設し、取得日・説明・アイコンを表示。
- バッジ一覧ページ（例: `/kids/badges`）を新設し、過去獲得分を時系列で閲覧可能にする。
- バッジマスタに `junior_name/junior_description` と `standard_name/standard_description`（または i18n 的辞書）を追加。
- 一覧カード/詳細ページ/獲得演出の全てを `ageText` または同等の年齢分岐ヘルパーで統一。

**受け入れ基準**
- 子どもがホームのバッジをタップすると詳細が開く。
- 過去バッジを一覧で確認できる。
- junior/standardでバッジ文言が適切に切り替わる。

### L. バーコード画面の枠外でも即読み取りされる件
**確認結果**
- 現在のスキャナーは動画フレーム全体を検出対象としており、中央の四角枠は視覚ガイドのみ。
- そのため枠外のバーコードでも検出される実装。

**対応策**
- 検出領域を中央ROIに限定（videoをcanvasへ切り出してROI内のみ解析）。
- 同一ISBNの複数フレーム検出（例: 2〜3回連続一致）で確定し、誤検出を低減。
- 仕様文言をUIに明記（「枠の中に入れると読み取りやすい」）し、期待値を合わせる。

**受け入れ基準**
- 枠外のバーコード単発検出で即確定しにくくなる。
- ユーザーが「読み取り条件」をUIから理解できる。

## 3. 実機指摘ごとの解決策（既存項目）

### A. 大人モードログイン時、状態が分からない
**改善**
- 操作種別ごとに状態を分離（`signInLoading` / `signUpLoading`）。
- ログインボタンは「ログイン中…」+ スピナー + `aria-busy`。
- 失敗理由をユーザー向けに分類（認証失敗/通信失敗/予期せぬ失敗）。
- 連打防止（既存 `disabled` 維持）とフォーカス復帰を実装。

### B. パスワードを忘れたときのリカバリがない
**改善**
- 「パスワードを忘れた方」リンク追加。
- 再設定メール送信ページ + パスワード更新ページを追加（Supabase Auth の標準フロー）。
- メール未着ヘルプ（迷惑メール確認/再送間隔）をUIに明記。

### C. スマホで「〇〇のホーム」「ログアウト」が改行
**改善**
- `sm` 未満は2段構成（上段: タイトル、下段: ログアウト）に切替。
- タイトル領域に `min-w-0` とテキスト省略を適用。
- 必要に応じてログアウトをアイコンボタン化。

### D. 親メッセージの表記ゆれ（おとな/かぞく/おうちのひと）
**改善方針**
- こども向け統一語を **「おうちのひと」** に統一。
- `ageText` の junior/standard 双方でトーンを定義し、ハードコードを排除。

### E. 8歳なのに漢字表示（仕様調査を踏まえた対応）
**改善手順（必須順）**
1. **設定確認**: 該当 child の `birth_year` と `age_mode_override` を確認。
2. **文言監査**: `/kids/home` `/kids/messages` `/kids/records` の対象ラベルに `ageText` 適用。
3. **junior文言修正**: 漢字が残る junior 文言をひらがなへ修正。
4. **回帰確認**: standard 側の表示が崩れていないか確認。

### F. 色味不一致（青リンク/青カード）
**改善**
- こどもUI用トークンを定義（`kid-link`, `kid-card`, `kid-accent`）。
- 「こどもホームへもどる」リンクを既存アンバー系へ寄せる。
- 読書記録カード背景を淡い紙色（生成り）へ変更。

### G. 読書記録カードをノート/スクラップブック風に
**改善（段階導入）**
- Phase 1: 紙色背景 + 角丸 + 軽い影
- Phase 2: マスキングテープ風装飾（疑似要素）
- Phase 3: 日付スタンプ/見出しのみ手書き風
- 常時: 画像依存を避けCSS中心でパフォーマンス維持
