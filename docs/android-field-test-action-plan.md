# Android実機テスト指摘への改善計画（仕様調査反映版）

## 0. 先に結論（対応順序）
1. **認証UX（大人ログイン）**: ログイン中表示の明確化 + パスワード再設定導線
2. **スマホ表示崩れ修正**: 「〇〇のホーム」「ログアウト」の改行対策
3. **年齢適応文言の不足箇所修正**: 8歳（junior）でも漢字が残る箇所を ageText 分岐へ統一
4. **用語統一**: 「おとな/家族/おうちのひと」表記の一本化
5. **配色統一**: 青リンク/青カードのトーン調整
6. **質感改善**: こども読書記録カードをノート/スクラップブック風へ段階的に改修

---

## 1. 仕様調査（ひらがな化は年齢切替仕様か？）

### 調査結果
- 年齢表示モードは `AgeMode = 'junior' | 'standard'`。
- 判定ロジックは以下:
  - `age_mode_override` が `junior` / `standard` の場合はそれを優先
  - `auto` の場合は `birth_year` から年齢算出し、**8歳以下なら junior**
  - 生年不正/未設定は `standard` にフォールバック
- 文言切替は `ageText(mode, { junior, standard })` で実装済み。

→ したがって、**「ひらがな化は年齢に応じて切り替える」が現行仕様**。

### 今回の指摘との関係
- 「8歳なのに漢字で表示される」は、仕様がないのではなく、
  1) 該当文言に `ageText` が未適用
  2) `junior` 側の文言が漢字のまま
  3) child の `birth_year`/`age_mode_override` 設定が想定と違う
  のいずれか（または複合）で発生する可能性が高い。

## 2. 実機指摘ごとの解決策

### A. 大人モードログイン時、状態が分からない
**現状**
- ボタン文言は `loading` 中に「処理中…」へ変化するが、サインイン/サインアップで共通状態。
- エラーは同一メッセージ領域に表示される。

**改善**
- 操作種別ごとに状態を分離（`signInLoading` / `signUpLoading`）。
- ログインボタンは「ログイン中…」+ スピナー + `aria-busy`。
- 失敗理由をユーザー向けに分類（認証失敗/通信失敗/予期せぬ失敗）。
- 連打防止（既存 `disabled` 維持）とフォーカス復帰を実装。

**受け入れ基準**
- 押下直後から完了まで、ログイン進行状態が明確に判別できる。

### B. パスワードを忘れたときのリカバリがない
**現状**
- ログイン画面に再設定導線がない。

**改善**
- 「パスワードを忘れた方」リンク追加。
- 再設定メール送信ページ + パスワード更新ページを追加（Supabase Auth の標準フロー）。
- メール未着ヘルプ（迷惑メール確認/再送間隔）をUIに明記。

**受け入れ基準**
- サポート介入なしで、メール認証経由の自己復旧が可能。

### C. スマホで「〇〇のホーム」「ログアウト」が改行
**現状**
- ヘッダーが `flex items-center justify-between` の1段固定で、狭幅時に折返し。

**改善**
- `sm` 未満は2段構成（上段: タイトル、下段: ログアウト）に切替。
- タイトル領域に `min-w-0` とテキスト省略を適用。
- 必要に応じてログアウトをアイコンボタン化。

**受け入れ基準**
- 360px幅で見出し・操作ボタンが重ならず、改行崩れが発生しない。

### D. 親メッセージの表記ゆれ（おとな/かぞく/おうちのひと）
**改善方針**
- こども向け統一語を **「おうちのひと」** に統一。
- `ageText` の junior/standard 双方でトーンを定義し、ハードコードを排除。

**受け入れ基準**
- ホーム・メッセージ一覧・関連CTAで同一語彙に統一。

### E. 8歳なのに漢字表示（仕様調査を踏まえた対応）
対象:
- 最近読んだ本
- 過去の記録を見る
- 家族からのメッセージ
- すべて見る
- 既読
- 未読

**改善手順（必須順）**
1. **設定確認**: 該当 child の `birth_year` と `age_mode_override` を確認（autoで8歳判定になるか）。
2. **文言監査**: `/kids/home` `/kids/messages` `/kids/records` の対象ラベルが `ageText` で分岐しているか確認。
3. **junior文言修正**: 漢字が残る junior 文言をひらがなへ修正。
4. **回帰確認**: standard 側の既存表示が崩れていないか確認。

**推奨 junior 文言**
- 最近読んだ本 → `さいきん よんだ ほん`
- 過去の記録を見る → `まえの きろくを みる`
- 家族からのメッセージ → `おうちのひと からの メッセージ`
- すべて見る → `ぜんぶ みる`
- 既読 → `よんだ`
- 未読 → `まだ よんでない`

### F. 色味不一致（青リンク/青カード）
**改善**
- こどもUI用トークンを定義（`kid-link`, `kid-card`, `kid-accent`）。
- 「こどもホームへもどる」リンクを既存アンバー系へ寄せる。
- 読書記録カード背景を淡い紙色（生成り）へ変更。

### G. 読書記録カードをノート/スクラップブック風に
**改善（段階導入）**
- Phase 1: 紙色背景 + 角丸 + 軽い影
- Phase 2: マスキングテープ風装飾（疑似要素）
- Phase 3: 日付スタンプ/見出しのみ手書き風
- 常時: 画像依存を避けCSS中心でパフォーマンス維持
