# カラーバランス仕様（2026-03）

## 目的

- 子どもモード（junior）と大人モード（standard）の視覚的な分断を減らし、同じ「家族向け読書体験」として連続性を持たせる。
- 既存の可読性を保ちながら、全体の配色を「暖色ベース + 補助寒色」に再整理する。

## 方針

1. **大人モードの色調を子どもモードへ寄せる**
   - kids 配下では age mode に依存せず、暖色系トーン（amber/orange/rose）を基調に統一する。
   - standard だけが寒色・無彩色に寄る状態を避け、文字量の差で年齢差を表現する。

2. **全体バランスの再設計**
   - アプリ全体のベース背景を `slate` から `amber` 系に変更し、kids画面とのトーン差を縮小する。
   - 主要アクションは orange、通知は rose、補助情報は violet/sky を低彩度で利用。

## トークン運用（Tailwind系）

- ベース背景: `bg-amber-50`
- メインアクション: `bg-orange-500 hover:bg-orange-600`
- セカンダリアクション: `border-amber-300 bg-amber-50 text-amber-800`
- 通知/注意: `rose-50/rose-200/rose-700`
- 情報補助: `violet-50/violet-200`
- カード面: `bg-white/95 + border-amber-100`

## 画面別適用ルール

- `/kids/layout`
  - 背景を暖色グラデーション化（amber + rose）して、ページ間の印象を統一。
- `/kids/home`
  - ナビカード、通知カード、一覧カードを暖色中心に寄せる。
  - 未読ありのみ rose を強調色として利用。
- `/kids/messages`
  - 未読・既読とも amber ベースで表示、既読アクションボタンは orange。
- `/kids/calendar`
  - 当日強調を blue から orange へ変更し、月ナビも暖色化。
- `/kids/records`（本棚コンポーネント）
  - タブのアクティブ色を amber 系濃色から orange に変更し、背景グラデーションを柔らかく調整。

## アクセシビリティ

- テキストは `text-amber-900` 以上を基本としてコントラストを確保。
- バッジ・通知など色だけに依存する箇所は、絵文字・ラベルを併記して意味を補完。

## 今後の拡張

- 親向け画面にも同等トークンを段階適用し、ブランドカラーとして一本化する。
- Tailwind クラス直書きを段階的にデザイントークン化し、保守性を高める。
