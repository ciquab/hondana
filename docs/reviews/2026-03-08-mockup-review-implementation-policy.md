## ✅ 実装する（モックが示す改善・新機能）

### モックで確認できた「現状と差分のある要素」のみ対象とする

#### 1. 子ども記録フォームの読了状況表現（Step 3-C の延長）

| モックの表現 | 実装内容 |
|---|---|
| 「📖 さいごまで読んだ！」/ 「🔖 とちゅうまで読んだ」の 2 択ボタン | ステータス選択を子ども向け文言（`よんだ` / `とちゅう`）に対応づけ |

現状の `status` カラム（`reading` / `completed` / `want_to_read`）は変えず、フォームの選択肢の見せ方を子ども語に合わせるだけ。

#### 2. 子ども切り替えタブ（親ダッシュボード）（Step 5-5）

| 項目 | 内容 |
|---|---|
| 機能 | ダッシュボードに子ども別タブを設け、ページ遷移なしで子どもを切り替え |
| 必要な変更 | `app/dashboard/page.tsx` の状態管理（useState + タブ UI） |
| DB 変更 | なし（既存データをフィルタリングするだけ） |

モックが現状の「子ども一覧リスト」形式を示しているため、タブ実装はあくまで UX 改善として追加。既存の `/children/[childId]` ルートは残す。

#### 3. 登録 FAB（子どもホーム）（Step 5-6）

| 項目 | 内容 |
|---|---|
| 機能 | 子どもホーム画面にフローティングボタンを追加し「スキャン → 記録入力」を 1 タップで開始 |
| 必要な変更 | `app/kids/home/page.tsx` に FAB コンポーネント追加。既存 2×2 ナビグリッドは残す |
| ルーティング | `/kids/records/new` への遷移。既存ルートを再利用 |

現状モックが示す 2×2 カードナビは廃止しない。FAB はそれに加える形で追加。

#### 4. 親からのおすすめ送信（Step 5-2）

| 項目 | 内容 |
|---|---|
| 機能 | 親が本を検索 → 子どもの「よみたいほん」リストに追加 |
| 必要な変更 | `reading_records` に `suggested_by` カラム追加 or 専用テーブル。親 UI に「おすすめする」ボタン |
| 子ども側 | 子どもホームに「おとなからのおすすめ」セクションを追加 |

## 🎨 見た目（スタイリング）の方針

モックの配色・ボタン形状は大部分が Step 3.5/3.6 で実装済み。

| 要素 | モックの指定 | 実装の状況 |
|---|---|---|
| こどもナビカード | emerald/indigo/violet/rose 4 色 | ✅ 一致（`app/kids/home/page.tsx`） |
| スタンプ色ハイライト | 🌟amber / 😊emerald / 😐slate / 😓violet | ✅ 一致（Step 3.6-2） |

| バッジピル | `bg-amber-100 rounded-full` | ✅ 一致 |

### 実装する（小規模・見た目のみの差分）

| 要素 | モックの指定 | 現状 | 対応方針 |
|---|---|---|---|
| フォント | `-apple-system, 'Hiragino Sans', sans-serif` | 未指定（ブラウザデフォルト） | `app/globals.css` の `body` に `font-family` 追加 |
| body 背景色 | `bg-slate-100` | `bg-slate-50` | `globals.css` を `bg-slate-100` に変更（微差だが統一） |
| 表紙なし本のプレースホルダー | 絵文字入りグラデーション（`bg-gradient-to-b from-sky-300…`） | グレー無地（`bg-slate-100 text-slate-400`） | タイトル文字をそのまま表示するスタイルに揃える（絵文字は不要・タイトル短縮テキストで対応） |

| 要素 | 判断 |
|---|---|
